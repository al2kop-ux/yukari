Executive Summary

LoanAgainstAssets.co.uk is a young but promising asset-backed lending and pawnbroking business, FCA-regulated and operating from Canary Wharf, London. The site is currently indexed by Google but has near-zero organic visibility — ranking only for its own brand name and the tail variation "loan against property" at position 89–103. The domain authority is negligible, the site title is still the default WordPress "My Blog", and virtually every on-page SEO signal is either missing or misconfigured. However, the business model sits in an underserved niche with low-to-medium keyword difficulty on many high-value asset loan terms, meaning rapid wins are achievable within 3–6 months with focused effort.

Biggest Quick Win

Fix site title, add H1s, create dedicated service pages for each asset type (watches, gold, jewellery, cars). These pages can rank within 60–90 days for low-competition terms.

Biggest Growth Lever

"Pawnbroker near me" has 8,100 monthly UK searches. GMB optimisation + location landing pages targeting East London boroughs is the highest-volume local opportunity.

Biggest Risk

Section 01

Technical SEO Audit

🚨
Site Title is "My Blog" — This is a P0 Emergency The site title and meta title showing as "My Blog: Home" in Google SERPs is the single most damaging issue on this site. It signals low quality to both Google and users, will crush CTR, and must be fixed before anything else.
🔴 Site Name = "My Blog" (WordPress Default) Critical

The WordPress site name is set to "My Blog" — this appears as the title in Google SERPs, browser tabs, and social shares. All ranked pages show "My Blog: Home" or "My Blog - [Page Name]". This destroys brand recognition and CTR.

Fix: WordPress Admin → Settings → General → Change "Site Title" to "Loan Against Assets UK | FCA-Regulated Asset Loans | LoanAgainstAssets.co.uk" and Tagline to "Fast, Discreet Loans Against Luxury Watches, Jewellery, Gold, Cars & Property."
🔴 No Individual Page Meta Titles or Descriptions Critical

No pages have unique, keyword-optimised meta titles or descriptions set. Google is auto-generating snippets from body content — all currently pulling from navigation menus.

Fix: Install Yoast SEO or RankMath. Set unique <60 char titles and <160 char meta descriptions for every page. Each service page title should follow: [Asset Type] Loan UK | No Credit Check | LoanAgainstAssets
🔴 Homepage H1 = "Home" (Semantically Worthless) Critical

The homepage H1 tag contains only the word "Home" — this is a WordPress navigation label being rendered as the page heading. Google uses the H1 as a primary relevance signal.

Fix: Change homepage H1 to: "Asset-Backed Loans in the UK — Borrow Against Your Valuables Today" or similar keyword-rich heading. Every page must have exactly one H1.
🔴 No Structured Data / Schema Markup Critical

Zero Schema markup detected on any page. For a financial services firm, LocalBusiness, FAQPage, Service, and BreadcrumbList schema are essential for SERP rich results and AI Overview citations.

Fix: Implement: LocalBusiness schema on homepage (with FCA reg number, address, phone, opening hours), FAQPage schema on FAQ page, Service schema on each asset loan page, BreadcrumbList on all pages. Use RankMath or a custom plugin for WordPress.
🟠 No XML Sitemap in Google Search Console (likely) High

While sitemaps exist at /page-sitemap1.xml, /post-sitemap1.xml and /category-sitemap1.xml, there is no evidence a sitemap index has been submitted to GSC. Category sitemap suggests WordPress categories with no real content.

Fix: Create a sitemap index file at /sitemap.xml pointing to all three sitemaps. Submit to Google Search Console and Bing Webmaster Tools. Remove empty category pages from sitemap or set to noindex.
🟠 Local Landing Pages Have No Unique Content High

The site has 25+ location pages (Berkshire, Canary Wharf, Greenwich etc.) but all appear to be thin/duplicate content pages with no localised copy, local schema, or unique service descriptions. These will trigger Google's thin content/spam filters.

Fix: Either (a) add 600+ words of localised, unique content to each priority location page OR (b) noindex/remove low-priority location pages and consolidate into a well-structured local landing page strategy (see Section 7).
🟠 No robots.txt or Canonical Tags Verified High

No evidence of proper robots.txt directives or canonical tags. Duplicate content risk from www/non-www, HTTP/HTTPS, and URL parameter variations.

Fix: Verify robots.txt at /robots.txt, set canonical to HTTPS non-www (or www consistently), and ensure all pages have self-referencing canonicals via SEO plugin.
🟠 Core Web Vitals — Unknown (Likely Poor) High

The site uses multiple large WhatsApp image uploads (PNG with "removebg" in filenames) as the logo. These unoptimised images combined with WordPress defaults likely mean poor LCP scores. Mobile performance is a ranking factor.

Fix: Run PageSpeed Insights. Compress and convert all images to WebP. Use lazy loading. Consider a caching plugin (WP Rocket or LiteSpeed Cache). Target LCP <2.5s, CLS <0.1, INP <200ms.
🔵 Internal Linking is Minimal Medium

Pages do not link to each other using keyword-rich anchor text. Blog posts don't link to relevant service pages. The link equity and topical authority signals are lost.

Fix: Build an internal linking map. Every blog post should link to at least 2 relevant service pages. Service pages should cross-link. Use descriptive anchor text (e.g., "loan against a Rolex" not "click here").
🔵 Missing Open Graph & Twitter Card Tags Medium

No OG tags detected. Social shares will appear with generic previews, hurting social traffic CTR.

Fix: Add og:title, og:description, og:image to all pages via Yoast/RankMath. Create a branded OG image template (1200×630px) for blog posts.
